# .hviz Feature File Specification

Version: 0.1  
Generated by: `backend/process_audio.py`  
Consumed by: `frontend/src/hooks/useFeatureFrame.ts`

---

## Overview

A `.hviz` file is a JSON document produced offline by the Python audio pipeline.
It contains all musical features needed to drive the visualizer, pre-computed at
**30 fps** so the frontend only needs to look up a frame index — no DSP in the browser.

The companion TypeScript types live in `frontend/src/types/hviz.ts`.

---

## Top-level schema

```json
{
  "meta":     HvizMeta,
  "frames":   HvizFrame[],
  "segments": HvizSegment[]
}
```

---

## HvizMeta

| Field         | Type   | Description                                  |
|---------------|--------|----------------------------------------------|
| `title`       | string | Song title (from filename or ID3 tag)        |
| `duration_s`  | number | Total duration in seconds                    |
| `fps`         | number | Frames per second (always 30)                |
| `stem_sr`     | number | Sample rate of separated stems (Hz)          |
| `version`     | string | Spec version string, e.g. `"0.1"`            |

---

## HvizFrame

One object per frame. `t` is the frame's timestamp in seconds.

### Melody (vocals stem via CREPE)

```json
"melody": {
  "midi":       number | null,   // MIDI note 0–127, null = unvoiced
  "pitch_hz":   number | null,   // raw Hz before quantisation
  "conf":       number,          // CREPE confidence 0–1
  "vel":        number,          // RMS of vocals stem, normalised 0–1
  "attack":     number           // onset sharpness 0–1 (0 = legato, 1 = hard attack)
}
```

### Bass (bass stem via CREPE)

```json
"bass": {
  "midi":             number | null,
  "pitch_hz":         number | null,
  "octave":           number | null,   // midi // 12
  "pitch_class":      number | null,   // midi % 12
  "conf":             number,
  "vel":              number,
  "attack_sharpness": number           // drives SH snap speed 0–1
}
```

### Chord (vocals + other stem via librosa chroma → template match)

```json
"chord": {
  "root":    number | null,   // pitch class 0–11, null = unknown
  "quality": string | null,   // "maj" | "min" | "dom7" | "maj7" | "min7" |
                               // "dim" | "aug" | "hdim7" | "sus2" | "sus4"
  "conf":    number           // template match confidence 0–1
}
```

### Drums (drums stem — onset detection + bandpass per instrument)

```json
"drums": {
  "kick":     number,   // energy 0–1
  "snare":    number,
  "hihat":    number,
  "cymbal":   number,
  "softperc": number,
  "sharpness": {        // transient attack time per instrument 0–1
    "kick":     number,
    "snare":    number,
    "hihat":    number,
    "cymbal":   number,
    "softperc": number
  }
}
```

### Chroma (12-dim vector, normalised)

```json
"chroma": [number, number, ..., number]   // length 12, index = pitch class
```

### Spectrogram row (for Chladni floor shader)

```json
"spectrogram_row": [number, ...]   // 128 mel-scale bins, normalised 0–1
```

### Perceptual (Phase 7 MLP output — zeros until Phase 7)

```json
"perceptual": {
  "energy":  number,   // 0–1  → particle speed, bloom intensity
  "valence": number,   // 0–1  → warm/cool color temperature bias
  "tension": number,   // 0–1  → camera shake, field turbulence
  "density": number    // 0–1  → atmosphere particle count, mist thickness
}
```

---

## HvizSegment

Structural segments from librosa `agglomerative` segmentation.

```json
{
  "start": number,   // seconds
  "end":   number,   // seconds
  "label": string    // "intro" | "verse" | "chorus" | "bridge" | "outro" | "seg_N"
}
```

## Frame indexing in the frontend

```ts
const frameIndex = Math.round(audioContext.currentTime * meta.fps);
const frame = frames[Math.min(frameIndex, frames.length - 1)];
```

---

## Null handling

- `melody.midi`, `bass.midi`: `null` means unvoiced/silent — the helix should decay nodes, not error.
- `chord.root`, `chord.quality`: `null` when confidence < 0.4 — hold last known chord.
- All energy fields are `0.0` when the stem is silent (never null).
